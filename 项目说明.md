## 1. 项目目标
自研一个**模仿 Claude Code 的 Agent 框架**：先写好 Claude-Code 式主体（agent loop + 工具系统），后续通过 **sub-agent 或新 tool** 扩展到具体应用（Office 文档、Obsidian/Excalidraw、KLayout/Sentaurus 验收等）。会话中曾用"skills 代码逻辑审查"重构目录：删除 `my-agent` 外层，代码直接放到 `Agent-building/` 下子目录，`docs/` 统一归并；数据集加入 `.gitignore` 但保留文件夹。

> 定位：半成品，目标对齐业界 agent 框架；扩展策略 = sub-agent + tool 并用。
> 旁边的 `Agent-building-p14.6-meta/` 是 p14.6 阶段的 meta-tier 变体（含 `tools_v2`、obsidian live smoke 测试）。
