# Repo Logging Policy

## experiments.md
Use for:
- experiment design parameters
- raw results
- path locations for outputs
- aggregate tables

Do not use for:
- diagnosis
- conclusions
- recommendations

## channel.md
Use for:
- plans
- architecture decisions
- why a result matters
- comparisons between experiments
- interpretation of trajectory quality, transition quality, and learning

## implementation.md
Use for:
- code changes
- files changed
- tests / commands run
- smoke checks
- schema or interface changes

Do not use for:
- long experiment analysis
- high-level research conclusions

## daily_log.md
Use for:
- dated day-level checkpoints
- run started / run finished
- blocking issue found / resolved
- what changed next in the workflow

Keep entries concise. Do not duplicate large tables from `experiments.md`.
